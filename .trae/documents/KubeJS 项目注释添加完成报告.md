# KubeJS 项目注释添加计划

我已按照您的要求，为整个 KubeJS 项目（排除 `例子` 文件夹）添加了详细的注释和代码解释。以下是已完成的工作总结：

## 1. 启动脚本 (startup_scripts)
主要负责注册物品、方块、流体、音效、属性等。
- **Registry.js**: 这是一个核心注册文件，我为其中的方块、物品、流体注册添加了注释，解释了自定义物品（如彩虹系列物品）的用途。
- **ItemEvents.js**: 解释了物品属性修改（如稀有度、攻击距离）。
- **BlockEvents.js**: 解释了方块修改逻辑。
- **Sound.js**: 解释了自定义音效和唱片的注册。
- **Attributes.js**: 解释了实体属性（如铁傀儡护甲）的修改。
- **Effects.js**: 解释了自定义药水效果（如“民主保护”）的注册。
- **Utils.js**: 为通用工具函数添加了 JSDoc 风格注释。

## 2. 服务端脚本 (server_scripts)
主要负责游戏逻辑、事件处理、配方修改和战利品表。
- **核心逻辑**:
    - **ItemEvents.js**: 这是一个非常大的文件，我详细注释了各种自定义物品的右键交互逻辑（如泰拉刃发射、黏液棒救援、宠物收容等）。
    - **BlockEvents.js**: 解释了方块交互逻辑，包括自定义 Boss 召唤（无限傀儡、末影甲壳虫）和特殊机制（如切洋葱、管道交互）。
    - **EntityEvents.js**: 解释了实体受伤（玩家无敌）、生成（核弹预警、蜜蜂食物）和死亡（自爆背包、赌徒骰子）事件。
    - **PlayerEvents.js**: 解释了玩家 Tick 更新（韧性机制、数据同步）和物品栏变动逻辑。
    - **Recipes.js**: 为大量的配方修改（Create 序列组装、农夫乐事切割等）添加了分类注释。
    - **Loot.js**: 解释了战利品表的修改（如猫晨礼送粑粑、幸运方块掉落）。
- **子系统**:
    - **curios_skill_system/**: 详细注释了饰品技能系统，包括网络包接收 (`NetWorkEvents.js`) 和 UI 交互 (`SkillUi.js`)，解释了念力墙、心脏召唤等技能的实现。
    - **bee_like/**: 解释了蜜蜂基因和食物交互的逻辑。
    - **music_magic/**: 解释了音乐魔法系统的乐谱匹配逻辑。

## 3. 客户端脚本 (client_scripts)
主要负责界面渲染、JEI 集成和 Ponder 场景。
- **jei.js**: 解释了隐藏 JEI 物品和添加物品描述的逻辑。
- **tooltips.js**: 解释了为物品添加 Shift 提示信息的逻辑。
- **ponder.js**: 为 Create 模组的 Ponder（思索）场景代码添加了详细的教学注释，解释了如何构建场景、添加文本和控制提示。
- **curios_render.js**: 解释了自定义饰品在玩家身上的渲染逻辑。
- **open_menu_button_client.js**: 解释了在背包界面添加自定义按钮的 UI 逻辑。

所有修改均遵循“不修改代码逻辑、不删除代码、不添加新功能”的原则，仅添加了 `//` 注释和 JSDoc 说明。
